# Настройка системы Linux

Команда `tune` в GhostNet применяет все эти настройки автоматически.

## Автоматическое применение

```bash
sudo ./ghostnet tune
```

Предварительный просмотр без применения:
```bash
./ghostnet tune --dry-run
```

## Настройки sysctl

### Дескрипторы файлов

```
fs.file-max = 2097152
fs.nr_open  = 2097152
```

Каждое TCP-соединение использует один дескриптор файла. Лимит по умолчанию в Linux — 65536. При 10 000 одновременных соединений нужно минимум 10 000 + накладные расходы.

### Очередь TCP-соединений

```
net.core.somaxconn            = 65535
net.ipv4.tcp_max_syn_backlog  = 65535
```

Ограничивает количество соединений, ожидающих принятия. Должно быть достаточно большим для обработки всплесков трафика без потерь.

### Диапазон портов

```
net.ipv4.ip_local_port_range = 1024 65535
```

GhostNet открывает одно исходящее соединение на каждое клиентское. Расширение диапазона до 1024–65535 даёт ~64 000 одновременных исходящих соединений.

### Буферы сокетов

```
net.core.rmem_max     = 134217728
net.core.wmem_max     = 134217728
net.ipv4.tcp_rmem     = 4096 87380 134217728
net.ipv4.tcp_wmem     = 4096 65536 134217728
```

Большие буферы улучшают пропускную способность на высоколатентных или высокополосных соединениях.

### TIME_WAIT и повторное использование соединений

```
net.ipv4.tcp_tw_reuse     = 1
net.ipv4.tcp_fin_timeout  = 15
```

`tcp_tw_reuse` позволяет повторно использовать сокеты TIME_WAIT для новых исходящих соединений.

### Управление перегрузкой BBR

```
net.core.default_qdisc              = fq
net.ipv4.tcp_congestion_control     = bbr
```

BBR — алгоритм управления перегрузкой от Google. Обеспечивает более высокую пропускную способность и меньшую задержку по сравнению с CUBIC.

### rp_filter (обязателен для внедрения через raw-сокет)

```
net.ipv4.conf.all.rp_filter     = 0
net.ipv4.conf.default.rp_filter = 0
```

Фильтрация обратного пути отбрасывает пакеты, где исходный IP не соответствует ожидаемому интерфейсу. GhostNet внедряет пакеты с IP локальной машины в качестве источника — ядро отбросит их, если rp_filter включён. **Эта настройка обязательна для работы bypass-внедрения.**

## Настройки ulimit

Записываются в `/etc/security/limits.d/99-ghostnet.conf`:

```
*    soft nofile 1048576
*    hard nofile 1048576
```

Для текущей сессии:
```bash
ulimit -n 1048576
```

## Проверка

```bash
sysctl net.ipv4.tcp_congestion_control
ulimit -n
cat /proc/sys/net/core/somaxconn
sysctl net.ipv4.conf.all.rp_filter
getcap ./ghostnet
```
